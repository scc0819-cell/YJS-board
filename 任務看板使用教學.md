## 🚀 快速開始 <a name="快速開始"></a>

### 步驟 1：啟動看板系統

開啟終端機，執行：

```bash

```


## 🛠️ 五種使用方法 <a name="使用方法"></a>

### 方法 1：網頁看板（最直觀）⭐推薦

**適用情境**：日常查看、管理任務

1. 開啟瀏覽器：http://localhost:8080
2. 查看任務狀態
3. 點擊「➕ 新增任務」建立任務
4. 填寫表單：
   - 任務名稱（必填）
   - 描述（選填）
   - Agent（選填，預設 main）
5. 點擊「建立」

**優點**：
- ✅ 圖形化介面，直觀易懂
- ✅ 即時更新（每 10 秒自動重新整理）
- ✅ 可看到完整統計資訊

---

### 方法 4：Telegram 機器人

**適用情境**：行動裝置、即時通知

你的 Telegram bot 已經設定好了！

1. 在 Telegram 搜尋你的 bot
2. 發送訊息給 bot
3. bot 會回覆並記錄任務

**設定 Telegram 通知**（進階）：
```bash
# 設定排程任務透過 Telegram 通知
openclaw cron add \
  --cron "0 8 * * *" \
  --name "早晨檢查" \
  --agent main \
  --message "請執行早晨檢查" \
  --channel telegram \
  --to "你的 Telegram Chat ID"
```

**優點**：
- ✅ 隨時隨地接收通知
- ✅ 可用手機操作
- ✅ 即時推播

### 方法 5：OpenClaw 內建命令

**適用情境**：管理排程任務

```bash
# 新增排程任務
openclaw cron add \
  --cron "0 9 * * *" \
  --name "每日會議提醒" \
  --description "提醒參加每日會議" \
  --agent main \
  --message "提醒：9 點有每日會議"
```

```bash
# 停用排程任務
openclaw cron disable <任務 ID>
```

```bash
# 啟用排程任務
openclaw cron enable <任務 ID>
```

```bash
# 立即執行排程任務（測試用）
openclaw cron run <任務 ID>
```

```bash
# 刪除排程任務
openclaw cron rm <任務 ID>
```

**優點**：
- ✅ 官方工具，穩定可靠
- ✅ 完整功能支援
- ✅ 與 OpenClaw 深度整合

## 🎓 進階功能 <a name="進階功能"></a>

### 1. 自訂任務鏈規則

編輯檔案：`~/.openclaw/workspace/task-chain-rules.json`

**範例：加入自訂規則**
```json
{
  "rules": [
    {
      "name": "會議準備鏈",
      "trigger": "會議提醒",
      "next_tasks": [
        {
          "name": "準備會議資料",
          "description": "整理會議需要的文件和簡報",
          "delay_minutes": 0
        },
        {
          "name": "發送會議通知",
          "description": "提醒與會人員",
          "delay_minutes": 30
        }
      ]
    }
  ]
}
```

---

### 2. 條件式任務鏈

某些任務只在特定條件下才產生：

```json
{
  "rules": [
    {
      "name": "Git 提交鏈",
      "trigger": "Git 監控",
      "next_tasks": [
        {
          "name": "提交變更",
          "description": "提交未提交的檔案",
          "condition": "has_changes"
        }
      ]
    }
  ]
}
```

`condition: "has_changes"` 表示只有在 git 有變更時才產生任務。

---

### 3. 多 Agent 協作

設定不同任務使用不同 Agent：

```bash
# 建立任務時指定 Agent
curl -X POST http://localhost:8080/api/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "name": "複雜任務",
    "agent": "openai-codex/gpt-5.3-codex"
  }'
```

**Agent 選擇建議**：
- `main` (Qwen 3.5 397B)：一般任務
- `openai-codex/gpt-5.3-codex`：複雜推理
- `ollama/qwen3.5:cloud`：快速回應

---

### 4. 任務優先級

在任務描述中加入優先級標記：

```
名稱：【緊急】回覆客戶郵件
名稱：【重要】準備會議資料
名稱：【普通】整理文件
```

---

### 5. 任務標籤

使用標籤分類任務：

```

名稱：程式開發 #程式碼 #git
```

## ❓ 常見問題 <a name="常見問題"></a>

### Q1: 看板無法開啟怎麼辦？

**A:** 檢查服務是否運行
```bash
# 檢查 API 是否運行
ps aux | grep task-api

# 如果沒有運行，重新啟動
~/.openclaw/workspace/scripts/start-dashboard.sh
```

---

### Q2: 排程任務沒有自動執行？

**A:** 檢查 Gateway 狀態
```bash
# 查看 Gateway 狀態
openclaw status

# 重啟 Gateway
openclaw gateway restart

# 查看排程任務
openclaw cron list

# 測試執行
openclaw cron run <任務 ID>
```

---

### Q3: 任務鏈沒有自動產生新任務？

**A:** 檢查任務鏈處理器
```bash
# 檢查是否運行
ps aux | grep task-chain

# 查看日誌
cat /tmp/task-chain.log

# 重新啟動
pkill -f task-chain-handler.py
nohup python3 ~/.openclaw/workspace/scripts/task-chain-handler.py > /tmp/task-chain.log 2>&1 &
```

---

### Q4: 如何刪除任務？

**A:** 目前支援以下方式：
```bash
# 方法 1: 直接編輯任務檔案
nano ~/.openclaw/workspace/memory/task-board.json

# 方法 2: 使用 Python 腳本
python3 << 'EOF'
import json
with open('/home/yjsclaw/.openclaw/workspace/memory/task-board.json') as f:
    data = json.load(f)
# 刪除所有已完成任務
data['tasks'] = [t for t in data['tasks'] if t['status'] != 'completed']
with open('/home/yjsclaw/.openclaw/workspace/memory/task-board.json', 'w') as f:
    json.dump(data, f, indent=2, ensure_ascii=False)
print("已完成任務已清除")
EOF
```

---

### Q5: 如何備份任務資料？

**A:** 備份任務檔案
```bash
# 備份
cp ~/.openclaw/workspace/memory/task-board.json \
   ~/.openclaw/workspace/memory/task-board.json.bak.$(date +%Y%m%d)

# 還原
cp ~/.openclaw/workspace/memory/task-board.json.bak.20260228 \
   ~/.openclaw/workspace/memory/task-board.json
```

---

### Q6: 可以在手機上看嗎？

**A:** 可以！看板是響應式設計：
1. 確保電腦和手機在同一網路
2. 在手機瀏覽器輸入：`http://<電腦 IP>:8080`
3. 例如：`http://192.168.1.100:8080`

---

**A:** 
```bash
# 1. 確認 Telegram 已連線
openclaw status | grep Telegram

# 2. 設定排程任務通知

  --cron "0 8 * * *" \
  --name "早晨檢查" \
  --agent main \

# 3. 測試通知
```

```bash
# 啟動看板
~/.openclaw/workspace/scripts/start-dashboard.sh

# 列出任務
~/.openclaw/workspace/scripts/task-tracker.sh list

# 新增任務
~/.openclaw/workspace/scripts/task-tracker.sh add "名稱" "描述"

# 查看統計
~/.openclaw/workspace/scripts/task-tracker.sh stats

# 查看排程
openclaw cron list

# 新增排程
openclaw cron add --cron "0 8 * * *" --name "任務名稱" --agent main

# API 測試
curl http://localhost:8080/api/tasks
```
