## 📋 系統介紹

這是一套為 OpenClaw 設計的**永久記憶系統**，解決 AI 記憶斷裂問題，實現：

1. ✅ **記憶連續性** - 跨 Session 保持脈絡不中斷
2. ✅ **智能檢索** - 隨時調用相關歷史討論
3. ✅ **自動關聯** - 相關話題自動連結
4. ✅ **無縫注入** - 新 Session 自動載入相關記憶

---

## 🎯 解決的問題

### ❌ 之前的問題
```
Session 1: 討論了 Agent 團隊架構
Session 2: 完全不記得 Session 1 的內容
Session 3: 需要重新說明所有背景
→ 對話破碎、片斷化、效率低落
```

### ✅ 現在的解決方案
```
Session 1: 討論 Agent 團隊 → 自動儲存記憶
Session 2: 自動載入 Agent 相關記憶 → 延續討論
Session 3: 自動關聯所有相關話題 → 完整脈絡
→ 對話連續、智能檢索、效率提升
```

## 🚀 快速開始

### 1️⃣ 新增記憶

**命令列方式：**
```bash
cd ~/.openclaw/workspace

# 新增決策記憶
python3 scripts/memory_manager.py add "決定建立 6 人 Agent 團隊" decision "自動化系統，Agent 團隊"

# 新增事實記憶
python3 scripts/memory_manager.py add "集團有 5 家公司" fact "集團架構"

# 新增偏好記憶
python3 scripts/memory_manager.py add "喜歡藍色漸層設計風格" preference "設計風格"
```

**在對話中自動採集：**
```
系統會自動從對話中提取重要內容並儲存
（需要設定自動採集規則）
```

## 📊 記憶分類說明

### 記憶類型

| 類型 | 說明 | 範例 |
|------|------|------|
| **decision** | 重要決策 | 「決定採用 OpenClaw 內建記憶系統」 |
| **fact** | 事實資訊 | 「集團有 5 家公司」 |
| **preference** | 用戶偏好 | 「喜歡藍色漸層設計」 |
| **task** | 任務資訊 | 「建立任務管理看板」 |
| **relationship** | 關係資訊 | 「王經理負責昱金生能源」 |
| **general** | 一般記憶 | 「今天討論了自動化系統」 |

### 重要程度（1-10）

| 等級 | 說明 | 範例 |
|------|------|------|
| 10 | 關鍵決策 | 集團併購、重大投資 |
| 8-9 | 重要決策 | 系統架構、技術選型 |
| 5-7 | 一般記錄 | 日常討論、任務進度 |
| 1-4 | 次要資訊 | 臨時想法、備忘錄 |

## 🛠️ 進階功能

### 自動採集規則（開發中）

設定自動從對話中提取記憶的規則：

```python
# 範例：自動提取決策
if "決定" in message and ("要" in message or "採用" in message):
    capture_memory(message, type='decision')

# 範例：自動提取任務
if "建立任務" in message or "指派" in message:
    capture_memory(message, type='task')
```

### 定期摘要（開發中）

每週/每月自動生成記憶摘要：

```bash
# 每週一 09:00 自動執行
python3 scripts/memory_manager.py summary --period week
```

### 記憶導出（開發中）

導出記憶為不同格式：

```bash
# 導出為 Markdown
python3 scripts/memory_manager.py export --format md

# 導出為 JSON
python3 scripts/memory_manager.py export --format json

# 導出為 PDF
python3 scripts/memory_manager.py export --format pdf
```

self.workspace = Path.home() / '.openclaw' / 'workspace'

編輯 `scripts/memory_manager.py` 的 `capture_memory` 函數，新增自訂類型。

編輯 `scripts/memory_manager.py` 的 `_update_topics_index` 和 `_update_entities_index` 函數。

## 📊 效能優化

### 大量記憶管理

當記憶數量超過 1000 筆時：

1. **定期歸檔** - 將舊記憶移動到歸檔區
2. **摘要壓縮** - 將多筆記憶壓縮為摘要
3. **索引分片** - 按月建立獨立索引

### 檢索速度優化

1. **限制搜尋範圍** - 指定時間範圍
2. **使用主題索引** - 避免全文搜尋
3. **快取熱門結果** - 快取常用搜尋結果

## 🐛 常見問題

### Q1: 記憶沒有被儲存？
**A:** 檢查檔案權限和路徑是否正確

### Q2: 搜尋不到記憶？
**A:** 確認關鍵字是否正確，或嘗試主題搜尋

### Q3: 新 Session 沒有自動載入記憶？
**A:** 需要設定 session_injector.py 在 Session 開始時調用

### Q4: 記憶檔案太大？
**A:** 使用摘要功能壓縮舊記憶，或歸檔到獨立檔案
