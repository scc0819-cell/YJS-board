## 概述

### 目的

本文件定義昱金生能源日報系統在發生災難時的恢復流程，確保系統能在最短時間內恢復運作，減少業務損失。

### 適用範圍

- 日報系統伺服器
- 資料庫
- 附件檔案
- 使用者帳號與權限

---

## RTO 與 RPO

### RTO (Recovery Time Objective) - 恢復時間目標

| 系統組件 | RTO | 說明 |
|---------|-----|------|
| 日報系統 | 4 小時 | 系統恢復可用的最長時間 |
| 資料庫 | 2 小時 | 資料庫恢復的最長時間 |
| 附件檔案 | 24 小時 | 附件完全恢復的最長時間 |

### RPO (Recovery Point Objective) - 恢復點目標

| 資料類型 | RPO | 說明 |
|---------|-----|------|
| 資料庫 | 24 小時 | 最多損失 24 小時的資料 |
| 附件檔案 | 24 小時 | 最多損失 24 小時的上傳檔案 |
| 審計日誌 | 24 小時 | 最多損失 24 小時的操作紀錄 |

## 災難類型

### 等級 1 - 輕微（Local）

- 單一服務當機
- 資料庫損毀
- 部分檔案遺失

**應變**：IT 管理員現場處理

### 等級 2 - 中等（Site）

- 伺服器硬體故障
- 辦公室停電
- 網路中斷

**應變**：啟動備援伺服器，通知廠商

### 等級 3 - 嚴重（Regional）

- 火災、水災
- 地震
- 大規模停電

**應變**：啟動異地備份，臨時辦公室運作

## 應變流程

### 階段 1：發現與通報（0-15 分鐘）

1. **發現問題**
   - 系統監控警報
   - 用戶回報
   - 定期檢查發現

2. **初步評估**
   - 確認問題範圍
   - 判斷災難等級
   - 記錄發生時間

3. **通報**
   - 通知 IT 管理員
   - 通知相關主管
   - 啟動應變小組（等級 2 以上）

### 階段 2：應變與隔離（15-60 分鐘）

1. **隔離問題**
   - 停止受損服務
   - 防止問題擴大
   - 保護現場證據

2. **啟動備援**
   - 等級 1：本地修復
   - 等級 2：啟動備援伺服器
   - 等級 3：啟動異地備份

3. **持續監控**
   - 監控修復進度
   - 記錄所有操作
   - 定期回報狀態

### 階段 3：恢復與驗證（1-4 小時）

1. **資料恢復**
   - 從備份還原資料庫
   - 還原附件檔案
   - 驗證資料完整性

2. **系統恢復**
   - 重啟服務
   - 測試基本功能
   - 確認用戶可訪問

3. **驗證測試**
   - 登入測試
   - 資料查閱測試
   - 附件上傳/下載測試

### 階段 4：復原與檢討（1-7 天）

1. **根本原因分析**
   - 調查事故原因
   - 撰寫事故報告
   - 提出改善建議

2. **系統復原**
   - 修復受損硬體
   - 更新系統軟體
   - 強化安全措施

3. **檢討與改進**
   - 召開檢討會議
   - 更新應變流程
   - 加強教育訓練

## 還原步驟

### 資料庫還原

```bash
# 1. 停止服務
pkill -f "python3 app.py"

# 2. 還原資料庫
cp /path/to/backup/app_YYYYMMDD.db /mnt/c/Users/YJSClaw/Documents/Openclaw/daily_report_server/data/app.db

# 3. 驗證資料庫
sqlite3 /mnt/c/Users/YJSClaw/Documents/Openclaw/daily_report_server/data/app.db "SELECT COUNT(*) FROM users;"

# 4. 重啟服務
cd /mnt/c/Users/YJSClaw/Documents/Openclaw/daily_report_server
nohup python3 app.py > /tmp/daily_report.log 2>&1 &

# 5. 測試訪問
curl http://localhost:5000
```

### 附件還原

```bash
# 1. 解壓縮備份檔
unzip /path/to/backup/attachments_YYYYMMDD.zip -d /tmp/restore/

# 2. 複製到目標目錄
cp -r /tmp/restore/* /mnt/c/Users/YJSClaw/Documents/Openclaw/daily_report_attachments/

# 3. 設定權限
chmod -R 755 /mnt/c/Users/YJSClaw/Documents/Openclaw/daily_report_attachments/

# 4. 驗證檔案數量
find /mnt/c/Users/YJSClaw/Documents/Openclaw/daily_report_attachments/ -type f | wc -l
```

### 完整系統還原

使用自動化腳本：

```bash
# 還原資料庫
./restore_from_backup.sh /path/to/app_YYYYMMDD.db

# 還原附件
./restore_from_backup.sh /path/to/attachments_YYYYMMDD.zip
```
