## 任務指派流程

1. **接收任務** — 從董事長指令或 cron 排程觸發
2. **分析任務** — 判斷由哪個 Agent 處理最合適
3. **Spawn 子代理** — 用 `sessions_spawn` 啟動對應 Agent
4. **記錄到看板** — 更新 `task-board.json` 狀態為 running
5. **等待完成** — 子代理完成後自動回報
6. **反饋處理** — 根據結果決定是否產生後續任務
7. **更新看板** — 標記完成，記錄到 memory

## Agent 指派規則

| 任務類型 | 指派 Agent | 說明 |
|----------|-----------|------|
| 系統監控、異常 | Argus | git 狀態、服務健康 |
| 報表、文件 | Scribe | 日報、PDF、摘要 |
| 排程、提醒 | Chronos | cron 管理、行事曆 |
| 數據分析 | Athena | 趨勢、統計、決策支援 |
| 郵件、通知 | Hermes | 訊息發送、郵件檢查 |
| 綜合/複雜 | Jenny | 需要協調多 Agent 的任務 |

## 反饋迴圈規則

- 子代理完成 → 更新 task-board.json
- 如果結果需要後續動作 → 自動產生新任務
- 如果發現問題 → 升級到 Jenny 處理
- 所有狀態變更 → 寫入 memory
