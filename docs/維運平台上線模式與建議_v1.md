# 昱金生維運管理平台｜上線模式與建議 v1

更新時間：2026-03-01 05:33 (GMT+8)

## 一、上線目標
- 不中斷既有 AUO 使用習慣
- 先「可監測、可追溯、可告警」，再逐步自動化處置
- 以低風險滾動上線，避免一次性大改造成誤報/干擾

---

## 二、建議上線模式（分 4 階段）

## Phase 0：影子模式（3~5 天）
**目的：先驗證，不影響現場流程**
- 每小時抓 88 場快照
- 規則引擎跑 P1/P2/P3，但僅寫檔不通知（dry-run）
- 每天產出「誤報/漏報檢討表」

**驗收門檻**
- 告警準確率 > 80%
- 不出現大量重複告警（同站同事件每小時最多 1 次）

## Phase 1：半自動上線（1~2 週）
**目的：先讓管理層有感**
- 開啟 Telegram 每小時摘要（P2/P3）
- P1 即時通知，但僅「通知+建議」，不自動派工
- 晚間總結 PDF 固定輸出

**驗收門檻**
- P1 漏報為 0
- P2 通知噪音（無效警報）< 25%

## Phase 2：工單聯動（2~4 週）
**目的：從告警進入管理閉環**
- P1/P2 事件自動建立工單
- 加入 SLA 倒數與升級規則（逾時升級）
- 看板顯示：待處理 / 處理中 / 已恢復 / 已結案

**驗收門檻**
- SLA 達標率 > 90%
- 平均修復時間（MTTR）持續下降

## Phase 3：智慧優化（持續）
**目的：降低誤報、提高預測能力**
- 導入連續命中去抖動（2/3 次命中才升級）
- 加入案場 offset 與季節性校正
- 週報加入「重複故障根因分析」

**驗收門檻**
- 告警準確率 > 90%
- 重複故障率逐月下降

---

## 三、三種上線策略（給董事長選）

1. **穩健型（推薦）**
   - 先 Phase 0→1，兩週後再到 Phase 2
   - 優點：風險最低、團隊容易適應

2. **加速型**
   - 一週內直接進 Phase 1 + 部分 Phase 2
   - 優點：成果快；缺點：初期告警噪音較高

3. **保守型**
   - 長期停留 Phase 0/1（只做監測與報告）
   - 優點：不影響作業；缺點：管理效益釋放慢

---

## 四、建議採用（我的建議）
建議採 **穩健型**：
- 第 1 週：Phase 0（影子模式）
- 第 2 週：Phase 1（通知上線）
- 第 3 週：Phase 2（工單聯動）

原因：目前資料品質已有「時間戳與容量異常點」需先校正，先驗證再放大可避免誤報影響信任。

## 六、風險與對策
1. **夜間低發電誤報**
   - 對策：夜間時段切換規則（不以發電落差判 P1）
2. **舊案場資料延遲導致噪音**
   - 對策：資料品質白名單/黑名單機制
3. **告警過多**
   - 對策：同站同事件去重 + 1 小時彙整
4. **人員負荷過高**
   - 對策：先通知後派工，逐步自動化
