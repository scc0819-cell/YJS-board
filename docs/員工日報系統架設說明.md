# 昱金生能源集團 - 員工日報系統架設與使用說明

## 📋 系統概述

這是一套完整的線上日報系統，包含：
1. **前端網頁** - 員工填寫日報
2. **後端伺服器** - 儲存與管理資料
3. **自動分析** - AI 分析、風險評估、任務指派
4. **管理報表** - Excel 報表自動生成

---

## 🚀 快速架設步驟

### 步驟 1：安裝必要套件

```bash
# 進入工作目錄
cd /home/yjsclaw/.openclaw/workspace

# 安裝 Python 套件
pip3 install flask flask-cors pandas openpyxl --break-system-packages
```

### 步驟 2：啟動伺服器

```bash
# 啟動後端伺服器
cd /home/yjsclaw/.openclaw/workspace/server
python3 app.py
```

伺服器啟動後會顯示：
```
🏢 昱金生能源集團 - 員工日報系統
📁 資料儲存路徑：/home/yjsclaw/.openclaw/workspace/daily_reports
🌐 伺服器網址：http://localhost:5000
👥 員工人數：10
```

### 步驟 3：讓員工訪問

#### 方案 A：區域網路訪問（推薦）

伺服器會自動監聽所有網路介面，員工可透過以下方式訪問：

```
http://[您的 IP]:5000
```

例如：`http://192.168.1.100:5000`

#### 方案 B：使用 ngrok 對外公開

```bash
# 安裝 ngrok
sudo snap install ngrok

# 啟動 ngrok
ngrok http 5000
```

會獲得一個公開網址，例如：`https://abc123.ngrok.io`

#### 方案 C：部署到公司伺服器

將整個 `server` 資料夾上傳到公司伺服器，使用 Nginx + Gunicorn 部署。

## 📱 員工使用流程

### 1. 訪問系統

員工開啟瀏覽器，前往系統網址。

### 2. 選擇員工

在首頁點擊自己的名字旁的「填寫日報」按鈕。

### 3. 填寫日報

填寫以下內容：

#### 基本資訊
- 姓名（自動帶入）
- 日期（預設今日）
- 部門（自動帶入）

#### 一、今日工作（必填）
- 案件編號（如：58-A）
- 案件名稱（如：成功國小）
- 工作類型（設計/現勘/會議/行政/施工/驗收/其他）
- 工作內容（具體描述）
- 進度%（0-100）
- 工時（小時）
- 產出物（如：圖面 v3、現勘報告）
- 狀態（已完成/進行中/待確認）

#### 二、明日計畫（必填）
- 案件編號
- 案件名稱
- 預計工作內容
- 預計工時
- 需要支援

#### 三、風險事項（有則必填）
- 案件編號
- 風險等級（🔴高/🟠中/🟢低）
- 風險說明
- 影響層面
- 需要協助

#### 四、今日總結（選填）
- 最大收穫
- 需要改進
- 其他備註

#### 五、附件
- 勾選附件類型（現場照片/會議記錄/文件檔案/其他）
- 附件說明

### 4. 提交

點擊「📤 提交日報」按鈕。

## 🤖 AI 自動分析流程

### 觸發條件

當日報提交率達 **80%** 時，自動觸發 AI 分析。

### 分析內容

1. **日報品質評分**
   - 完整性（A/B/C/D 級）
   - 進度標記準確性
   - 風險通報主動性

2. **風險評估**
   - 彙整所有高風險事項
   - 分析影響層面
   - 建議處理方案

3. **績效分析**
   - 計算各維度分數
   - 案場進度追蹤
   - 員工表現排名

4. **任務指派**
   - 自動生成待辦事項
   - 指派負責人
   - 設定追蹤日期

### 分析結果儲存

```
/daily_reports/2026-02-28/analysis_result.json
```
